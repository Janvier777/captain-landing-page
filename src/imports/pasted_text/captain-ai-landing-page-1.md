PART 2: ANIMATION SPEC (save for code phase)
Reference doc for when you're ready to build in HTML/CSS/JS

Animation Library Choice: AOS (Animate On Scroll) + Custom CSS
AOS is the simplest library for scroll-triggered animations — one script tag, data attributes on HTML elements, done. Custom CSS keyframes handle the floating blobs and glassmorphism. No React or complex JS needed.

Effect 1: Floating Gradient Blobs
What: 5-6 large, blurred gradient circles that drift slowly and continuously across the page, creating a living, breathing background.
CSS approach:
.blob {
  position: absolute;
  border-radius: 50%;
  filter: blur(80px);
  opacity: 0.5;
  animation: float 20s ease-in-out infinite;
  pointer-events: none;
}

@keyframes float {
  0%, 100% { transform: translate(0, 0) scale(1); }
  25% { transform: translate(30px, -40px) scale(1.05); }
  50% { transform: translate(-20px, 20px) scale(0.95); }
  75% { transform: translate(40px, 30px) scale(1.02); }
}
Placement plan:

Blob 1: Hero upper-right, 500px, peach-to-lavender, animation-duration 25s
Blob 2: Hero lower-left, 350px, coral-to-orange, animation-duration 18s, delay -5s
Blob 3: Problem section left, 400px, lavender-to-indigo, animation-duration 22s, delay -8s
Blob 4: Workflow section right, 300px, teal-to-lavender, animation-duration 20s, delay -12s
Blob 5: CTA section center, 450px, peach-to-coral, animation-duration 24s, delay -3s

Key details:

Each blob gets a different animation-duration and animation-delay so they don't sync up (feels organic)
Use mix-blend-mode: soft-light or normal depending on background
z-index: -1 so they stay behind content
Will-change: transform for GPU acceleration


Effect 2: Glassmorphism
What: Frosted glass cards with real backdrop blur, catching the colors of the gradient blobs behind them.
CSS approach:
.glass-card {
  background: rgba(255, 255, 255, 0.25);
  backdrop-filter: blur(20px);
  -webkit-backdrop-filter: blur(20px);
  border: 1px solid rgba(255, 255, 255, 0.3);
  border-radius: 20px;
  box-shadow: 0 8px 32px rgba(59, 54, 128, 0.08);
}
Key details:

The magic of glassmorphism is that it interacts with the floating blobs — as blobs drift behind cards, the frosted color shifts subtly. This creates a LIVING feel.
Use different opacity levels for different card types:

Feature cards: rgba(255,255,255,0.25)
Stat cards: rgba(255,255,255,0.35)
Nav bar: rgba(255,255,255,0.6)
Workflow step cards: rgba(255,255,255,0.2)


The blur amount matters: 20px for main cards, 12px for smaller elements, 30px for the nav


Effect 3: Scroll-Triggered Section Reveals
What: Each section fades and slides into view as the user scrolls down. Elements within sections stagger — title first, then subtitle, then cards one by one.

CDN: <link href="https://unpkg.com/aos@2.3.1/dist/aos.css" rel="stylesheet">
CDN: <script src="https://unpkg.com/aos@2.3.1/dist/aos.js"></script>
Init: AOS.init({ duration: 800, easing: 'ease-out-cubic', once: true, offset: 80 });

Alternating left/right creates a zigzag reveal that feels dynamic.
Connecting line animation:
.timeline-line {
  position: absolute;
  left: 50%;
  top: 0;
  width: 2px;
  height: 100%;
  background: linear-gradient(to bottom, #B8A9D4, #7BBAB5);
  transform-origin: top;
  transform: scaleY(0);
  transition: transform 1.5s ease-out;
}

.timeline-line.visible {
  transform: scaleY(1);
}

window.addEventListener('scroll', () => {
  const nav = document.querySelector('.nav');
  if (window.scrollY > 100) {
    nav.classList.add('scrolled');
  } else {
    nav.classList.remove('scrolled');
  }
});

Performance Notes

Use will-change: transform on blobs and animated elements
Limit blobs to 5-6 maximum (more tanks mobile performance)
Use backdrop-filter sparingly on mobile — test on real devices
Consider reducing blob count or disabling blob animation on mobile via @media query
AOS "once: true" prevents re-triggering, which saves performance
Total page weight should stay under 500KB excluding images
